GitHub Actions — Quick Revision Cheatsheet

Workflows & Events

  • Workflow file: .github/workflows/<name>.yml.
  • Triggers (on:): push, pull_request, workflow_dispatch, issues.

Jobs & Steps

  • Job: jobs.<id>, runs on a runner (runs-on), use needs to depend.
  • Step: use uses: (action) or run: (shell).

Artifacts

- uses: actions/upload-artifact@v5
  with:
    name: build-artifact
    path: dist

- uses: actions/download-artifact@v5
  with:
    name: build-artifact
    path: dist

Cache npm

- uses: actions/setup-node@v3
  with:
    node-version: 18
    cache: 'npm'
- run: npm ci

Run local tools

- run: npm exec vite -- build
# or
- run: npx vite build

Job outputs & env

  • Set output in a step: echo "name=value" >> $GITHUB_OUTPUT.
  • Expose job output: use jobs.<id>.outputs and read via needs.<id>.outputs.<name>.
  • Persist env: echo "MY=val" >> $GITHUB_ENV. Access in shell as $MY.

Environment Variables & Secrets

  • Scopes: workflow-level env, job-level env, step-level env, and secrets (tokens/credentials).
  • Set during run: echo "MY_VAR=hello" >> $GITHUB_ENV (persists for later steps).
  • Set step output: echo "result=ok" >> $GITHUB_OUTPUT (for job outputs).
  • Access: ${{ env.MY_VAR }} or ${{ secrets.MY_SECRET }} in expressions; $MY_VAR in shell.

Controlling Workflow & Job Execution

  • Triggers: on: push|pull_request|workflow_dispatch|schedule|issues.
  • Run on paths: use paths / paths-ignore under triggers.
  • Conditionals: use if: on jobs/steps, e.g. if: github.event_name == 'push'.
concurrency:
  group: ${{ github.ref }}
  cancel-in-progress: true

Jobs & Docker Containers

jobs:
  test:
    runs-on: ubuntu-latest
    container:
      image: node:18
    steps:
      - run: node -v

Use services: for linked containers (databases) and options: for docker runtime flags.

Building & Using Custom Actions

  • Action types: JavaScript, Docker, Composite (YAML).
  • Action metadata is in action.yml (defines inputs/outputs and runs).
name: 'My Action'
inputs:
  msg:
    required: true
outputs:
  result:
runs:
  using: 'node12'
  main: 'dist/index.js'

Use actions via uses: owner/repo@v1 or local with uses: ./path. Pin versions and avoid leaking secrets.
